# Cursor Top 10 チートシート

印刷またはピン留め用の 1 枚まとめ。

## 使い分け早見表

| やりたいこと | 使う機能 | ショートカット |
|-------------|---------|---------------|
| コードの続きを書いてほしい | Tab 補完 | `Tab` |
| 選択範囲を書き換え | Inline Edit | `Ctrl+K` |
| コードを理解したい（変更なし） | Ask モード | `Ctrl+L` → Ask |
| 機能を実装してほしい | Agent | `Ctrl+I` |
| 大きな変更の計画 | Plan Mode | Agent 内 `Shift+Tab` |
| 特定ファイルを渡す | @ メンション | `@filename` |
| 毎回同じ指示 | Rules | `.cursor/rules/` |
| 定型ワークフロー | Skills | `SKILL.md` + `/skill名` |
| 外部サービス連携 | MCP | Settings → MCP |
| 品質・コスト調整 | モデル選択 | Agent パネル上部 |

## プロンプトの型

### 理解（Ask）
```
@sample-app/src/app.js
このファイルのデータの流れを説明して。変更はしないで。
```

### 小さな修正（Inline Edit）
```
（関数を選択して Ctrl+K）
この関数に JSDoc コメントを追加して
```

### 機能追加（Agent）
```
@sample-app/src/app.js
Todo に「編集」機能を追加して。ダブルクリックでインライン編集できるように。
```

### 大きな変更（Plan → Agent）
```
（Plan Mode で）
Todo アプリにフィルター機能を追加する計画を立てて。
影響ファイル、実装ステップ、テスト方法を含めて。
```

## @ メンション一覧

| 記法 | 意味 |
|------|------|
| `@filename` | 特定ファイル |
| `@folder/` | フォルダ全体 |
| `@Commit` | 未コミット差分など |
| `@Branch` | ブランチ差分 |
| `@Terminals` | ターミナル出力 |
| `@Browser` | 組み込みブラウザのコンテキスト |
| `@Past Chats` | 過去の会話を参照 |

Web 検索やコードベース探索は `@Web` / `@Codebase` を付けず、自然言語で依頼する（Agent がツールで実行）。

## Rules vs Skills

| | Rules | Skills |
|---|-------|--------|
| いつ読み込む | 毎回自動 | 必要なときだけ |
| 用途 | コーディング規約 | 定型ワークフロー |
| 置き場所 | `.cursor/rules/` | `.cursor/skills/` または `~/.cursor/skills-cursor/` |

## 安全チェックリスト（変更をマージする前）

- [ ] diff を目視で確認した
- [ ] 意図しないファイルが変更されていない
- [ ] ブラウザ / テストで動作確認した
- [ ] シークレット（API キー等）が含まれていない

## 困ったとき

1. プロンプトを短く・具体的にする
2. `@file` で範囲を絞る
3. Ask モードで現状を理解してから Agent
4. Plan Mode で大きな変更を分割する
