# 演習 05: Plan Mode で計画

**所要時間:** 25分  
**使う機能:** Plan Mode

## 目的

大きな変更の前に **Plan Mode** で実装計画を立て、承認してから構築フェーズに進む習慣を身につける。

## 背景

Agent にいきなり「リファクタして」と頼むと、10 ファイルを一気に変更されることがあります。Plan Mode では**計画書が先に出て、承認後に実行**されます。

## 手順

### Step 1: Plan Mode に入る

1. **Ctrl + I** で Agent を開く
2. **Shift + Tab** で Plan Mode に切り替え（またはモード選択 UI から Plan）

### Step 2: 計画を依頼

```
@sample-app/

Todo アプリを「コンポーネント分割」したい。

現状 app.js に HTML 生成・イベント・ストレージが全部入っている。
以下を満たすリファクタ計画を立てて:

1. storage.js — localStorage の読み書き
2. ui.js — DOM 生成・更新
3. app.js — 初期化とイベント配線のみ

計画には含めて:
- 変更するファイル一覧
- 各ステップの順序
- 各ステップ後の動作確認方法
- リスク（壊れやすいポイント）

まだコードは変更しないで。計画だけ。
```

### Step 3: 計画をレビュー・編集

Plan が表示されたら:

- ステップが多すぎないか？
- テスト方法は現実的か？
- 不要なファイル変更はないか？

修正したい点があればチャットで:

```
計画の Step 2 と 3 を統合して。まず storage.js だけ切り出して動作確認してから ui.js に進みたい。
```

### Step 4: 計画を承認して実行

計画に納得したら **Build / Execute**（UI のボタン）で構築フェーズへ。

Agent が計画に沿って段階的に変更します。

### Step 5: 各ステップで確認

計画どおりに進んでいるか、各ステップ後に:

```powershell
start sample-app\src\index.html
git diff --stat
```

## 成功条件

- [ ] Plan Mode で計画書を生成した
- [ ] 計画を 1 回以上修正した
- [ ] 承認後に構築が実行された
- [ ] リファクタ後も Todo アプリが動作する

## Plan Mode が向いているタスク

- 3 ファイル以上の変更
- リファクタリング
- 新機能 + テスト追加
- 不明なコードベースへの初介入

## 向いていないタスク

- 1 行の typo 修正 → Inline Edit
- 「この関数の意味は？」→ Ask
