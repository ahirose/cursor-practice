# 演習 03: Ask モードでコード理解

**所要時間:** 20分  
**使う機能:** Chat（Ask モード）

## 目的

コードベースを**一切変更せず**、AI に質問して理解する Ask モードを使いこなす。

## 背景

いきなり Agent に「直して」と頼む前に、Ask で現状を把握すると精度が上がります。Ask モードではファイルの読み取りのみ行われ、編集はされません。

## 手順

### Step 1: Ask モードに切り替え

1. **Ctrl + L** でチャットパネルを開く
2. 入力欄の左または上部で **Agent** → **Ask** に切り替え

### Step 2: ファイルについて質問

次のプロンプトをそのまま送ってください:

```
@sample-app/src/app.js

このファイルのデータフローを説明して。
- todos 配列はどこで更新される？
- localStorage との同期タイミングは？
- イベントリスナーは何個ある？

変更はしないで。
```

### Step 3: コードベース全体を探索

`@Codebase` は不要です。Ask / Agent に自然言語で頼めばコードベースを検索します:

```
このプロジェクトで「Todo の完了状態」はどこで管理されている？
関連ファイルをリストアップして。コードベースを検索して答えて。
```

### Step 4: 改善点を聞く（まだ変更しない）

```
@sample-app/src/app.js @sample-app/src/styles.css

現状の Todo アプリの UX 上の問題点を 3 つ挙げて。
コード変更はまだしないで。
```

回答をメモ帳に保存しておき、後の演習で使います。

### Step 5: 特定の行について深掘り

`app.js` の `render()` 関数内の任意の行にカーソルを置き、**Add to Chat**（行番号横のアイコン、または右クリック）でその行だけコンテキストに追加して質問:

```
この行は DOM 操作のどの部分？パフォーマンス上の懸念はある？
```

## 成功条件

- [ ] Ask モードで 4 つ以上の質問をした
- [ ] `@file` 指定と、指定なしのコードベース探索の両方を試した
- [ ] ファイルが自動変更されていない（git diff が空、または自分の手動変更のみ）

## 確認方法

```powershell
git diff
```

Ask モードでは AI によるファイル変更はないはずです。
