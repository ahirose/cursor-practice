# 演習 02: Inline Edit (Ctrl+K)

**所要時間:** 20分  
**使う機能:** Inline Edit

## 目的

1 ファイル内の**選択範囲**を、チャットを開かずにサッと編集する Inline Edit に慣れる。

## 背景

Inline Edit は「この関数だけ」「この数行だけ」を直したいときの最適解です。Agent ほど重くなく、Tab 補完より意図を伝えやすいです。

## 手順

### Step 1: 関数をリファクタリング

1. `sample-app/src/app.js` を開く
2. `loadTodos` 関数全体を**マウスで選択**
3. **Ctrl + K** を押す
4. プロンプト欄に入力:

```
エラーハンドリングを強化して。localStorage が使えない場合（プライベートブラウズ等）は空配列を返し、console.warn で理由をログ出力して。
```

5. **Accept** で変更を適用

### Step 2: JSDoc を追加

1. `saveTodos` 関数を選択
2. **Ctrl + K**
3. プロンプト:

```
JSDoc コメントを追加。パラメータと副作用（localStorage への書き込み）を記載。
```

### Step 3: スタイルの微調整

1. `sample-app/src/styles.css` を開く
2. `.todo-item` ルールを選択
3. **Ctrl + K** → プロンプト:

```
ホバー時に背景色が #f9fafb になる transition 付きのスタイルを追加
```

### Step 4: 差分を確認

Inline Edit 適用後、エディタ上に diff が表示されます。**緑（追加）と赤（削除）** を必ず目視確認してから Accept してください。

## 成功条件

- [ ] Ctrl+K で 3 箇所以上編集した
- [ ] 各編集前に diff を確認した
- [ ] `app.js` がブラウザで引き続き動作する（`index.html` を開いて確認）

## よくある失敗

| 問題 | 対策 |
|------|------|
| 選択範囲が広すぎて意図しない変更 | 関数単位で選択を絞る |
| 英語のコメントが混ざる | プロンプトに「コメントは日本語で」と追加 |
