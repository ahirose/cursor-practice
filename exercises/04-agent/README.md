# 演習 04: Agent で機能実装

**所要時間:** 30分  
**使う機能:** Agent モード

## 目的

Agent に複数ファイルの編集と動作確認を任せ、**指示 → レビュー → 修正** のサイクルを回す。

## 背景

Agent はターミナルコマンドの実行、複数ファイルの同時編集、エラー発生時の再試行ができます。日常開発で最も「力を発揮する」モードです。

## 事前準備

- Auto-run terminal commands が **OFF** であることを確認（最初はコマンド実行前に確認したい）

## 手順

### Step 1: シンプルな機能追加

**Ctrl + I** で Agent を開き、次を送信:

```
@sample-app/src/app.js @sample-app/src/index.html @sample-app/src/styles.css

Todo アプリに「すべて完了 / 未完了のみ / 完了済みのみ」のフィルター UI を追加してください。

要件:
- 3 つのボタンまたはタブで切り替え
- フィルター状態は URL や localStorage に保存しなくて OK
- 既存の追加・削除・完了機能は壊さない
- 変更後、sample-app/src/index.html をブラウザで開いて動作確認する方法を教えて
```

### Step 2: 出力をレビュー

Agent が diff を提案したら:

1. **変更されたファイル一覧**を確認（意図しないファイルがないか）
2. 各 diff を目視
3. 問題なければ Accept / 問題あれば「〇〇は触らないで」と追加指示

### Step 3: 動作確認

```powershell
start sample-app\src\index.html
```

フィルターが動くか手動テスト。

### Step 4: バグがあれば修正依頼

例:

```
フィルターで「完了済み」を選んでも、追加した新しい Todo が表示されない。未完了フィルター中は新規 Todo を表示するように修正して。
```

### Step 5: （発展）ターミナルで簡易サーバー

Agent に依頼:

```
sample-app を npx serve か python -m http.server でローカルサーバー起動する方法を教えて。CORS 問題があればそれも説明して。
```

## 成功条件

- [ ] Agent でフィルター機能が追加された
- [ ] diff をレビューしてから Accept した
- [ ] ブラウザで手動テストした
- [ ] 1 回以上、追加指示で修正した（任意だが推奨）

## 安全のコツ

- 「変更してよいファイル」を `@` で明示
- 大きな依頼は Plan Mode（演習05）に分割
- コミット前に `git diff` で全体確認
