# 演習 06: @ メンションでコンテキスト指定

**所要時間:** 20分  
**使う機能:** @ mentions

## 目的

プロンプトに**必要な情報だけ**を渡す `@` メンションの使い方をマスターする。

## 背景

コンテキストを渡しすぎると精度が下がり、渡し少ないと AI が勝手に探しに行って時間がかかります。`@` で「これだけ見て」と指定するのがコツです。

どのファイルが関係するか分からないときは `@` を無理に付けず、Agent に検索させて構いません。

## @ メンション一覧（演習で使うもの）

| 記法 | 例 | 用途 |
|------|-----|------|
| ファイル | `@sample-app/src/app.js` | 特定ファイルをコンテキストに |
| フォルダ | `@sample-app/src/` | フォルダ内全体（`/` でさらに掘れる） |
| Git | `@Commit` | 未コミット差分（Diff of Working State）など |
| ブランチ | `@Branch` | main などとのブランチ差分 |
| ターミナル | `@Terminals` | ターミナル出力を渡す |
| ブラウザ | `@Browser` | 組み込みブラウザのコンテキスト |

過去の会話を参照したいときは `@Past Chats` も使えます。

### 使わなくなった記法（参考）

Cursor 2.0 以降、Agent が自分でコンテキストを集められるため、次はメニューから外れています。

| 旧記法 | いまのやり方 |
|--------|-------------|
| `@Web` | 「Web で調べて」と自然言語で指示（Agent の Web 検索ツール） |
| `@Docs` | 公式・ライブラリ docs も自然言語で調べさせる（Editor では `@Docs` が残る場合あり） |
| `@Codebase` | 「コードベースを検索して」と聞く（Agent が自動検索） |
| `@Git` | `@Commit` / `@Branch` に置き換え |

## 手順

### Step 1: ファイル指定 vs 指定なし

**A（ファイル指定あり）** — Agent または Ask:

```
@sample-app/src/app.js

render 関数のループ内で createElement を何回呼んでいる？
```

**B（指定なし）** — 同じ質問を `@` なしで送る。

→ 回答速度・正確さの違いを体感。

### Step 2: フォルダ指定

```
@sample-app/src/

このフォルダ内のファイル間の依存関係を図示して。import/export は使っていないので、script タグと関数呼び出しで追って。
```

### Step 3: Git diff を渡す

1. 適当に `app.js` を 1 行編集（コメント追加など）
2. チャットで `@Commit` と打ち、一覧から **Diff of Working State** を選ぶ
3. 次を送る:

```
@Commit

今の変更に問題はある？コミット前レビューして。
```

### Step 4: Web 検索（`@Web` は不要）

自然言語で依頼すれば、Agent が Web 検索ツールを使います:

```
Cursor の Rules はどこに置く？ファイル形式は？
2026年時点の .cursor/rules 推奨形式も、公式ドキュメントや信頼できる最新情報を Web で調べて教えて。
```

### Step 5: 複数ファイルの組み合わせ

```
@sample-app/src/app.js @sample-app/src/styles.css

削除ボタンのスタイルが目立ちすぎ。app.js の deleteBtn 生成部分と styles.css を見て、控えめなデザインに変更して。styles.css だけ変更して。
```

## 成功条件

- [ ] `@file` `@folder` `@Commit`（または `@Branch`）のうち 3 種類以上使った
- [ ] Web 調査を `@Web` なしの自然言語で依頼した
- [ ] 「変更してよいファイル」を `@` とプロンプトで明示した
- [ ] 指定なし vs ありの違いを体感した

## ベストプラクティス

```
✅ @sample-app/src/app.js の render 関数を最適化
❌ プロジェクト全体を最適化して（範囲が広すぎ）

✅ @styles.css のみ変更。app.js は触らないで
❌ デザインを良くして（どのファイルか不明）

✅ Rules の置き場所を Web で最新情報から調べて
❌ @Web を付ける（メニューから削除済み）
```
